# Цифрология — сайт на Vercel

Готовый проект: сайт (React) + серверная функция-посредник, которая безопасно хранит твой ключ Anthropic.

**Как это устроено:** браузер → `/api/oracle` (там лежит ключ) → Anthropic.
Клиент ключ никогда не видит.

---

## Что нужно один раз установить
- **Node.js** (LTS) — https://nodejs.org
- Аккаунт на **Vercel** — https://vercel.com (бесплатно)
- Аккаунт на **GitHub** — https://github.com (удобнее всего)

---

## Шаг 1. Проверить локально (по желанию)
В папке проекта открой терминал и выполни:

```
npm install
npm run dev
```

Откроется адрес вида `http://localhost:5173`. ИИ-разборы локально не заработают,
пока не задан ключ (см. шаг 3) — но интерфейс, числа и матрица работают сразу.

## Шаг 2. Залить проект на Vercel
Самый простой путь:
1. Создай новый репозиторий на GitHub и загрузи туда все файлы этой папки.
2. На Vercel нажми **Add New → Project → Import** и выбери этот репозиторий.
3. Vercel сам поймёт, что это Vite, и соберёт сайт. Нажми **Deploy**.

(Альтернатива без GitHub: установи `npm i -g vercel`, затем в папке проекта выполни `vercel`.)

## Шаг 3. Вставить ключ (самое важное)
1. В проекте на Vercel открой **Settings → Environment Variables**.
2. Добавь переменную:
   - **Name:** `ANTHROPIC_API_KEY`
   - **Value:** твой ключ вида `sk-ant-...`
3. Сохрани и нажми **Redeploy** (Deployments → ⋯ → Redeploy), чтобы ключ подхватился.

Готово. Теперь сайт вызывает `/api/oracle`, а ключ остаётся только на сервере.

---

## Полезно знать
- **Экономия на ИИ:** в файле `api/oracle.js` можно сменить модель на
  `claude-haiku-4-5-20251001` — дешевле, но проще; `claude-sonnet-4-6` — качественнее.
- **Личный кабинет / история** сейчас хранится в браузере посетителя (localStorage).
  Это работает, но привязано к одному устройству. Для аккаунтов между устройствами
  позже понадобится база данных.
- **Оплата (Kaspi / карта):** кнопки в приложении — заглушки. Реальный приём оплаты
  подключается отдельно (Kaspi-ссылка/QR или эквайринг) и выдаёт доступ к премиуму.
- **Дисклеймер** про самопознание и развлечение оставь — он защищает тебя.

# Админка и база данных (кто зарегался / кто оплатил)

Чтобы видеть пользователей и оплаты, нужна общая база данных. Подключается в Vercel в пару кликов.

## Шаг A. Подключить базу (Upstash Redis) — бесплатно
1. В проекте на Vercel открой вкладку **Storage** (или **Integrations → Marketplace**).
2. Найди **Upstash** → **Redis** → нажми **Install / Add**.
3. Разреши Vercel создать базу (выбери бесплатный план). Он сам добавит переменные
   `KV_REST_API_URL` и `KV_REST_API_TOKEN` в проект.
4. Сделай **Redeploy** (Deployments → ⋯ → Redeploy).

## Шаг B. Задать пароль администратора
1. **Settings → Environment Variables** → добавь:
   - **Name:** `ADMIN_PASSWORD`
   - **Value:** любой твой пароль (например, `Astana2026!`)
2. **Redeploy**.

## Шаг C. Открыть админку
Зайди по адресу твоего сайта и добавь в конце **`#admin`**:
```
https://твой-сайт.vercel.app/#admin
```
Введи пароль. Внутри:
- список всех, кто зарегистрировался (имя, дата рождения, когда пришёл);
- кнопка **«отметить оплату»** у каждого — переключает статус «оплатил»;
- поле **«Ссылка Kaspi»** и цены — сохраняешь, и они появляются на кнопках оплаты.

## Как работает оплата (2 варианта)
В окне «Премиум» у клиента ровно два варианта: **разовая оплата через Kaspi** и **подписка**.
Оба ведут по твоей **ссылке Kaspi**. Схема:
1. Клиент жмёт оплату → открывается твоя Kaspi-ссылка → он платит.
2. Ты видишь платёж в приложении Kaspi.
3. Заходишь в `#admin` → находишь клиента → жмёшь **«отметить оплату»**.
4. У клиента premium открывается автоматически (кнопка «Я оплатил — проверить» или при следующем входе).

Ссылку Kaspi берёшь в приложении **Kaspi.kz → Бизнес → выставить счёт / оплата по ссылке**.

> Полностью автоматическое подтверждение оплаты (без ручной отметки) делается через
> Kaspi-эквайринг по договору — это следующий уровень, когда пойдёт поток клиентов.

# Секретный промокод (бесплатный полный доступ)

В админке (`#admin`) есть поле **«Секретный промокод»**. Впиши любое слово (например `VIP2026`)
и сохрани. Теперь любой, кто введёт этот код в окне оплаты, получит полный доступ бесплатно.

- Код проверяется на сервере и **наружу не отдаётся** — в коде сайта его не видно.
- Меняешь код в админке в любой момент — старый перестаёт работать.
- Удобно для друзей, блогеров, тест-доступа.

# Про подписку Kaspi (важно)
Ссылки Kaspi (оплата по ссылке/QR) — **разовые**, автосписания там нет.
Поэтому «подписка» работает как ежемесячная оплата по ссылке: клиент платит,
ты отмечаешь оплату в админке. Можно задать **отдельную ссылку для подписки**
(поле «Ссылка Kaspi для подписки»). Настоящее автопродление — только через
Kaspi-эквайринг по договору (следующий уровень).

# Английский язык и Apple Pay

**Языки.** В правом верхнем углу сайта — переключатель **RU / EN**. Он меняет интерфейс,
короткие описания и язык, на котором ИИ пишет все разборы и отвечает в чате. Выбор
запоминается в браузере. Так один и тот же сайт продаётся и русско-, и англоязычной аудитории.

**Apple Pay.** В окне «Премиум» есть кнопка Apple Pay. Сам по себе Apple Pay не принимает
деньги — он работает через процессор (обычно **Stripe**). Что сделать:
1. Заведи аккаунт на stripe.com, создай **Payment Link** (в нём включается Apple Pay).
2. В админке (`/#admin`) вставь эту ссылку в поле **«Ссылка Apple Pay / Stripe»** и сохрани.
Тогда кнопка Apple Pay будет вести на оплату. Для англоязычной аудитории это удобнее Kaspi.

**Промокод у клиента** вводится в окне «Премиум» → блок **«🎁 Есть промокод?»**.
Сам секретный код ты задаёшь в админке в поле **«🔑 Секретный промокод»**.

# Аккаунты клиентов (логин + пароль)

Теперь на входе клиент указывает **логин** (латиница, минимум 3 символа, уникальный).
- Логин **уже есть** → приложение просит **пароль** → вход.
- Логина **нет** → клиент придумывает пароль и вводит дату рождения → создаётся аккаунт.

Пароли хранятся в базе в виде хэша (SHA-256), не открытым текстом.

Требуется подключённый **Upstash Redis** (см. выше). Если базы нет — приложение
переходит в «гостевой» режим (имя + дата без пароля), чтобы сайт всё равно работал.

Необязательно: можно добавить переменную окружения **`AUTH_PEPPER`** (любая секретная
строка) — она усиливает хэш паролей. Если не задавать, используется значение по умолчанию.

В админке (`/#admin`) клиенты теперь показываются по их логину.

# БАЗА ДАННЫХ: TURSO (бесплатно, без карты) — заменяет Upstash

Серверные файлы в `api/` теперь работают с **Turso** (SQLite по HTTP). Настройка:

## Шаг 1. Создать базу
1. Зайди на **turso.tech** → зарегистрируйся (через GitHub или Google).
2. Создай базу данных (Create Database). Имя: `cifrologia`, план — бесплатный.

## Шаг 2. Взять URL и токен
1. Открой созданную базу.
2. Скопируй **Database URL** — выглядит как `libsql://cifrologia-xxxx.turso.io`.
3. Нажми **Create Token** (или Tokens → Generate) и скопируй **токен** — длинная строка.

## Шаг 3. Вставить в Vercel
Settings → Environment Variables, добавь две переменные:
- `TURSO_DATABASE_URL` = скопированный URL (`libsql://...`)
- `TURSO_AUTH_TOKEN` = скопированный токен

Проверь, что также заданы:
- `ANTHROPIC_API_KEY` — ключ ИИ
- `ADMIN_PASSWORD` — пароль для `/#admin`

## Шаг 4. Redeploy
Deployments → ⋯ → Redeploy.

Таблицы (`users`, `config`) создаются автоматически при первом обращении — вручную ничего создавать не нужно.

**Проверка:** создай на сайте имя+пароль+дату → выйди → введи то же имя с неверным паролем.
Если пишет «Такой профиль уже есть в базе — введи пароль», значит Turso подключён и работает. ✅
